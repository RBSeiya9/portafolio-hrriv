REQUEST LIFECYCLE

Flujo de Datos

Cómo viaja una petición desde el cliente Android hasta la base de datos y de vuelta.

01
📱

Client Request

El usuario realiza una acción en la app Android (ej: agregar al carrito). La app construye una petición HTTP/HTTPS hacia el servidor.

POST /api/v1/cart/add { "product_id": 12, "qty": 2 }
02
🔐

Auth & Validation

El API Gateway valida el token JWT del usuario, verifica su rol (cliente/admin) desde SharedPreferences y procede al procesamiento.

Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
03
⚙️

Business Logic

El servidor ejecuta la lógica de negocio: calcula subtotal, aplica ITBMS (7%), actualiza inventario y genera la factura.

subtotal = price * qty; total = subtotal * 1.07;
04
🗄️

Database Write

Se escribe en SQLite local y se replica vía polling al servidor cloud/VM Linux. Transacciones ACID garantizan integridad.

INSERT INTO orders (user_id, total, status) VALUES (?, ?, 'pending');
05
📧

SMTP Notification

El servicio SMTP envía automáticamente un correo de confirmación al usuario con el detalle de su pedido y factura.

SMTP → TLS:587 → "Tu pedido #4821 ha sido confirmado"
06
✅

Response

El servidor retorna una respuesta JSON al cliente Android con el estado de la operación y actualiza la UI en tiempo real.

200 OK { "order_id": 4821, "status": "confirmed" }
SECURITY LAYER

Capas de Seguridad

Protección multicapa implementada en los proyectos de RBSEIYA STUDIO.

🔒

HTTPS / TLS

Todas las comunicaciones están encriptadas con TLS 1.3. Certificados SSL en el API Gateway y servidor SMTP.

🛡️

Autenticación RBAC

Sistema de roles (Cliente/Admin) detectado automáticamente desde la base de datos tras el login. Sin acceso no autorizado.

🔑

SSH Hardened

Acceso al servidor Linux mediante claves SSH (key-based auth), sin contraseñas expuestas. Puerto 22 con fail2ban activo.

🧱

Firewall & NMAP Audit

Auditoría con NMAP para detección de puertos abiertos y vulnerabilidades en la red LAN inalámbrica del servidor.

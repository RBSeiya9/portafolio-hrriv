Sobre Mí

Ingeniero en formación,
enfocado en crear soluciones tecnológicas

Soy estudiante de la Licenciatura en Ingeniería en Sistemas y Computación en la Universidad Tecnológica de Panamá, Centro Regional de Chiriquí, donde actualmente curso el I Semestre 2026 bajo la asignatura Herramientas de Programación IV.

Me especializo en el desarrollo de aplicaciones móviles para Android utilizando Android Studio, integrando bases de datos locales con SQLite, diseño y administración de redes LAN mediante Cisco Packet Tracer, y la creación de interfaces intuitivas centradas en la experiencia del usuario. Además, cuento con experiencia en análisis, diseño e implementación de proyectos de ingeniería de software, aplicando buenas prácticas de desarrollo y documentación técnica.

Soy fundador de RBSEIYA STUDIO, una iniciativa de desarrollo tecnológico y servicios freelance donde transformo ideas en soluciones funcionales, aplicando estándares profesionales en planificación, cotización, diseño, desarrollo e implementación de software. Mi objetivo es desarrollar tecnología que combine funcionalidad, eficiencia y una experiencia de usuario de alta calidad, aportando valor real a cada proyecto.

REQUEST LIFECYCLE

Flujo de Datos

Cómo viaja una petición desde el cliente Android hasta la base de datos y de vuelta.

01
📱

Client Request

El usuario realiza una acción en la app Android. La app construye una petición HTTP/HTTPS hacia el servidor.

POST /api/v1/cart/add { "product_id": 12, "qty": 2 }
02
🔐

Auth & Validation

El API Gateway valida el token JWT del usuario, verifica su rol (cliente/admin) desde SharedPreferences.

Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
03
⚙️

Business Logic

El servidor ejecuta la lógica de negocio: calcula subtotal, aplica ITBMS (7%), actualiza inventario y genera la factura.

subtotal = price * qty; total = subtotal * 1.07;
04
🗄️

Database Write

Se escribe en SQLite local y se replica vía polling al servidor cloud/VM Linux. Transacciones ACID.

INSERT INTO orders (user_id, total, status) VALUES (?, ?, 'pending');
05
📧

SMTP Notification

El servicio SMTP envía automáticamente un correo de confirmación al usuario con el detalle de su pedido.

SMTP → TLS:587 → "Tu pedido #4821 ha sido confirmado"
06
✅

Response

El servidor retorna una respuesta JSON al cliente Android con el estado de la operación.

200 OK { "order_id": 4821, "status": "confirmed" }
SECURITY LAYER

Capas de Seguridad

Protección multicapa implementada en los proyectos de RBSEIYA STUDIO.

🔒

HTTPS / TLS

Todas las comunicaciones están encriptadas con TLS 1.3. Certificados SSL en el API Gateway y servidor SMTP.

🛡️

Autenticación RBAC

Sistema de roles (Cliente/Admin) detectado automáticamente desde la base de datos tras el login.

🔑

SSH Hardened

Acceso al servidor Linux mediante claves SSH (key-based auth), sin contraseñas expuestas. Puerto 22 con fail2ban.

🧱

Firewall & NMAP Audit

Auditoría con NMAP para detección de puertos abiertos y vulnerabilidades en la red LAN inalámbrica.

Mi experiencia en el semestre

Durante el desarrollo de este semestre, tuve la oportunidad de sumergirme en el diseño y creación de aplicaciones Android, la arquitectura de redes y el backend. Todos los trabajos y asignaciones, desde la aplicación de e-commerce hasta la configuración de servidores Linux, me permitieron consolidar mis conocimientos técnicos y enfrentar retos similares a los del entorno profesional.

Cada laboratorio y proyecto fue una pieza clave para entender cómo los componentes de software y hardware interactúan entre sí. Aprendí a diseñar interfaces de usuario modernas, estructurar bases de datos locales con SQLite y comunicar sistemas mediante protocolos seguros.

En conclusión, esta materia ha sido fundamental para mi formación como Ingeniero en Sistemas y Computación, otorgándome las herramientas necesarias para construir soluciones completas, robustas y escalables, preparándome para los desafíos futuros del desarrollo de software.
